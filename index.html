  • Relations:
    1. Relation1: List the system administrators and the number of configuration items they administer.
    2. Relation2: It is often the case that a system administrator will need to schedule maintenance for a server. As part of the scheduling process, s/he will need to know what other systems might be impacted. For a given configuration item (named 'super-server'), list all dependent configuration items.
    3. Relation 3: If an event occurs impacting the availability of a system, a system administrator will want to know what has changed both on that system and on other systems that support his/her system. For a given configuration item (named 'down-app'), list all configuration items that 'down-app' depends upon along with the last time they were changed and who the system administrator is for those configuration items.
    4. Relation 4: List configuration items by type.
    5. Relation 5: List the date, configuration item and summary of all recent changes ordered by date.
